# Datadex — Offline RAG MCP Tool for Hardware Engineers

**Datadex** is an offline Retrieval-Augmented Generation (RAG) MCP tool that ingests hardware datasheets and register files, then exposes them as queryable MCP tools for Claude Code.

---

## Architecture

```
+---------------------------------------+
|            Claude Code                 |
|  (asks questions via MCP protocol)     |
+------------------+--------------------+
                   | stdio (JSON-RPC)
+------------------v--------------------+
|         Datadex MCP Server             |
| datadex_search  datadex_register       |
| datadex_summary datadex_list_workspaces|
+----+---------------------------+------+
     |                           |
+----v----------+       +--------v---------+
|   ChromaDB    |       |     SQLite       |
| (vector search)|      |(register lookup) |
| all-MiniLM-L6 |       | name/addr index  |
+----+----------+       +--------+---------+
     |                           |
+----v---------------------------v---------+
|          Ingestion Pipeline              |
|  MarkdownParser (## split -> chunks)     |
|  PdfParser      (font-size headings)     |
|  DocxParser     (Heading styles)         |
|  ExcelParser    (register auto-detect)   |
+------------------------------------------+
```

## Project Structure

```
DataDex/
  .mcp.json                         MCP server registration (auto-generated by setup.ps1)
  setup.ps1                         One-command installer
  overview.md                       This file
  architecture.md                   Detailed technical architecture
  datadex/
    .venv/                          Python virtual environment (created by setup.ps1)
    datadex.py                      CLI: ingest, list, query
    datadex_server.py               MCP server entry point
    config.yaml                     Default configuration
    requirements.txt                Python dependencies
    core/
      ingestion/
        markdown_parser.py          .md -> heading-split chunks
        excel_parser.py             .xlsx -> structured register records
        pdf_parser.py               .pdf -> font-size heading chunks
        docx_parser.py              .docx -> Heading style chunks
      storage/
        chroma_store.py             ChromaDB wrapper (vectors)
        sqlite_store.py             SQLite wrapper (register lookup)
      query/
        engine.py                   Search + lookup orchestration
    workspaces/
      demo/                         One directory per project/product
        config.yaml                 Workspace config (auto-generated by setup.ps1)
        docs/                       Put your .pdf, .docx, .xlsx, .md files here
        data/                       Auto-created on ingest (chroma/ + registers.db)
```

## MCP Tools

| Tool | Parameters | Description |
|---|---|---|
| `datadex_search` | query, workspace? | Vector search - top-5 relevant chunks |
| `datadex_register` | register, workspace? | Structured register lookup by name/address |
| `datadex_summary` | topic, workspace? | Top-3 chunks as a summary |
| `datadex_list_workspaces` | -- | List all workspaces with counts |

## File Format Support

| Format | Parser | Chunk Strategy | Storage |
|---|---|---|---|
| `.md` (Markdown) | MarkdownParser | Split by ## / ### headings | ChromaDB |
| `.pdf` (PDF) | PdfParser | Font-size heading detection, else page-group | ChromaDB |
| `.docx` (Word) | DocxParser | Heading styles, else paragraph-group | ChromaDB |
| `.xlsx` / `.xls` (Excel) | ExcelRegisterParser | Auto-detect register columns | SQLite + ChromaDB |

## Workflow

### 1. Put your files in a workspace
```
datadex/workspaces/<name>/docs/
  ├── datasheet.pdf
  ├── register_map.xlsx
  └── specification.docx
```

### 2. Ingest

Ingest all workspaces at once:
```powershell
cd datadex
.\.venv\Scripts\python.exe datadex.py ingest --all
```

Or ingest a single workspace:
```powershell
.\.venv\Scripts\python.exe datadex.py ingest --workspace <name>
```

Add `--reindex` to clear and rebuild from scratch.

### 3. Use via Claude Code
Ask questions in English:
- *"Search Datadex for I2C timing parameters"*
- *"Look up register 0xFC in workspace demo"*
- *"Summarize what Datadex knows about power management"*

## Usage Tips

- All commands run from the `datadex/` directory
- Each workspace = one product or document set
- Use `--all` to ingest every workspace in one shot
- Use `--reindex` to clear and rebuild from scratch when docs change
- Adding a new workspace: add an entry to `$KnownWorkspaces` in `setup.ps1`, re-run it, drop docs in, then `ingest --all`
- No data leaves your machine — fully offline

- Also compatible with Linux / macOS (paths use forward-slash, no platform-specific calls)

### Runtime
| Requirement | Version | Purpose |
|---|---|---|
| **Python** | 3.10 – 3.14 | Runtime for Datadex. All dependencies ship pre-built wheels for Windows. |
| **pip** | 24.0+ | Package installer — comes with Python |
| **Claude Code** | latest | The AI assistant that queries Datadex via MCP protocol |
| **Git Bash** | optional | Used as terminal on Windows for Unix-style paths |

### Hard Disk Space
| Item | Size |
|---|---|
| Python installation | ~150 MB |
| `chromadb` + ONNX embedding model | ~200 MB (model auto-downloads on first ingest: `all-MiniLM-L6-v2`, ~80 MB) |
| pip package cache | ~100 MB |
| Your datasheets + Datadex databases | variable |
| **Total estimated** | **~500 MB minimum** |

### Network (one-time only)
- First `ingest` run downloads the `all-MiniLM-L6-v2` embedding model (~80 MB) from HuggingFace / Chroma CDN
- After that, **no internet connection required** — everything runs fully offline

### Memory (RAM)
- Minimal: ~256 MB (idle)
- During ingest: ~512 MB – 1 GB (for embedding computation)
- During search: ~256 MB
- ChromaDB keeps the model loaded in memory while running

| Package | Minimum Version | Install Size | Purpose |
|---|---|---|---|
| `chromadb` | 0.6.0 | ~120 MB (+ 80 MB model) | Vector database. Provides semantic search via ONNX-optimized `all-MiniLM-L6-v2` embedding model (~384-dim vectors). Auto-downloads the model on first use. |
| `openpyxl` | 3.1.0 | ~15 MB | Reads Excel `.xlsx` files. Parses register map spreadsheets — auto-detects columns by header name. |

| `python-docx` | 1.1.0 | ~2 MB | Reads Word `.docx` files. Iterates paragraphs and recognises "Heading N" styles. |

**Total pip install footprint: ~162 MB** (plus ~80 MB model auto-downloaded on first run)

## How to Install

### Prerequisites
- **Python 3.10+** — download from https://python.org or run `winget install Python.Python.3.14`
- **Claude Code** — `npm i -g @anthropic-ai/claude-code`

### Installation (one command)

```powershell
.\setup.ps1
```

That's it. The script will:
1. Auto-detect your Python installation
2. Create a `.venv` virtual environment inside `datadex/`
3. Install all dependencies into it
4. Write `.mcp.json` with the correct paths for your machine
5. Create all workspaces defined in `$KnownWorkspaces` with `docs/`, `data/`, and `config.yaml`

### Options

```powershell
.\setup.ps1 -NoVenv                  # install into global Python instead of a venv
.\setup.ps1 -Workspace myproject     # also create an extra custom workspace
```

To add a permanent new workspace, edit the `$KnownWorkspaces` table at the top of `setup.ps1` and re-run it.

### After setup

Drop your datasheets into a workspace `docs\` folder, then run:

```powershell
cd datadex
.\.venv\Scripts\python.exe datadex.py ingest --all
```

Then restart Claude Code — Datadex connects automatically.
